# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in IHW-ZoZ, please report it responsibly.

### How to Report

1. **DO NOT** create a public GitHub issue for security vulnerabilities
2. Use [GitHub Private Vulnerability Reporting](https://github.com/maplex18/IHW-ZoZ/security/advisories/new)
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

### What to Expect

- **Response Time**: We will acknowledge your report within 7 days
- **Updates**: We will provide updates on the progress of fixing the vulnerability
- **Disclosure**: Once the issue is resolved, we will coordinate with you on public disclosure

### Scope

This security policy applies to:
- The IHW-ZoZ desktop application
- Python backend components
- Electron/React frontend components

### Out of Scope

- Third-party dependencies (please report to their respective maintainers)
- Issues in development/testing environments only

## Security Best Practices

Since IHW-ZoZ processes files locally:
- Only process files from trusted sources
- Keep the application updated to the latest version
- Report any suspicious behavior immediately

# 安全政策

## 支援版本

| 版本    | 支援狀態           |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1   | :x:                |

## 回報漏洞

如果您發現 IHW-ZoZ 的安全漏洞，請負責任地回報。

### 如何回報

1. **請勿**在 GitHub 上建立公開的 Issue
2. 使用 [GitHub 私人漏洞回報功能](https://github.com/maplex18/IHW-ZoZ/security/advisories/new)
3. 請包含以下資訊：
   - 漏洞描述
   - 重現步驟
   - 潛在影響
   - 建議的修復方式（選填）

### 預期回應

- **回應時間**：我們將在 7 天內確認您的回報
- **進度更新**：我們會持續更新漏洞修復進度
- **公開揭露**：問題解決後，我們會與您協調公開揭露事宜

## 安全最佳實踐

由於 IHW-ZoZ 在本地處理檔案：
- 僅處理來自可信來源的檔案
- 保持應用程式更新至最新版本
- 如發現可疑行為請立即回報
